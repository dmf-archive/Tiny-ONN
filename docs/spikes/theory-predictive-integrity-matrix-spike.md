# Predictive Integrity Matrix Spike

## 摘要

- **探索目标**: 形式化定义 Predictive Integrity Matrix (PIM)，使“预测完整性”从标量门控升级为可承载知识结构保持的矩阵对象。
- **重要性**: 当前 PI 标量实验已表明，单独的 `λ_PI` 容易退化为 confidence-like 门控，无法充分表达结构保持需求。若 PIM 成立，它将为后续自蒸馏/持续学习实验提供新的几何载体。
- **决策截止日期**: 在编写首个 PIM 原型实验前完成，避免再次把结构问题误降维成标量加权问题。

## 研究问题

- **主要问题**: 如何构造一个既保留 PI“样本完整性筛选”作用、又保留 Fisher/协方差“结构保持”作用的矩阵对象。
- **次要问题**:
  - PIM 应定义在参数空间、表征空间，还是输出空间。
  - PIM 与 Fisher Matrix、Generalized Gauss-Newton、协方差矩阵之间的关系是什么。
  - Adam / ARS2-Neo 的二阶矩能否作为 PIM 的可计算代理。
  - 最小实验应验证“PIM 比标量 PI 更不容易退化为 confidence-like 门控”的哪一个侧面。

## Theory Brief

### Problem Statement

当前标量 PI 只能回答：

- teacher 是否值得信任；
- teacher 应参与多少；
- 蒸馏强度应大还是小。

但持续学习中的结构保持还需要回答：

- 哪些参数方向更敏感；
- 哪些子空间承载旧知识；
- 哪些更新方向更可能导致遗忘。

因此，若目标是保持知识结构，则需要从标量 `PI(x)` 提升到矩阵 `PIM`。

### Formal Specification

设：

- `π(x)` 为样本级 Predictive Integrity 标量；
- `G(x)` 为某个矩阵级结构对象；

则定义候选 Predictive Integrity Matrix：

`PIM = E_x [ π(x) · G(x) ]`

### 关键概念未定点

当前讨论中最需要先拆开的，不是公式细节，而是对象层级：

1. **输出梯度** 不是单一对象，至少应区分：
   - `J_x = ∂z/∂θ`：logits 对参数的 Jacobian，表示“参数如何影响输出”
   - `r_x = ∂ℓ/∂z`：loss 对 logits 的梯度，表示“输出当前偏离目标的方向与强度”
   - `g_x = ∂ℓ/∂θ = J_x^T r_x`：参数梯度，表示“实际更新方向”

2. **输出重要性** 与 **认知代价** 不是同一种二阶对象：
   - 输出重要性应定义在输出空间，如 `Λ_x = r_x r_x^T` 或 `H_z(x) = ∂²ℓ/∂z²`
   - 认知代价应定义在参数空间，如 `F_x` 或其二阶矩代理

3. 因此，PIM 不应直接写成“梯度 × loss × FIM”，而应分三步构造：
   - 先在输出空间定义偏离/重要性矩阵 `Λ_x`
   - 再通过 Jacobian 拉回参数空间：`S_x = J_x^T Λ_x J_x`
   - 最后再与认知代价矩阵 `C_x` 组合

4. **Adam 近似 Fisher** 的关键主要在二阶矩而非一阶动量：
   - `m_t` 主要提供方向平滑
   - `v_t` 才提供 `E[g²]` 风格的 Fisher-like 结构代理
   - 因而 `exp_avg_sq` 可被视为对角 Fisher 的经验近似，但不应与真 Fisher 混为同一物

5. 若引入 `JVP/HVP` 视角，则三类算子职责更清楚：
   - `JVP: J_x v` 回答“参数方向如何改变输出”
   - `VJP: J_x^T r_x` 回答“输出误差如何拉回参数空间”
   - `HVP: C_x v` 回答“该参数方向在几何/代价意义下有多贵”

这意味着，PIM 的核心问题不再是“是否要把 PI 标量乘上 Fisher”，而是：

**如何把输出偏差结构 `Λ_x`、参数-输出映射 `J_x`、认知代价矩阵 `C_x` 组织成一个闭合的二阶对象。**

PIM 至少应满足：

1. **半正定性**: `PIM ⪰ 0`
2. **结构性**: PIM 不能退化为单纯标量倍数单位阵，除非额外假设各向同性高斯。
3. **门控性**: 高 PI 样本对 PIM 的贡献更大，低 PI 样本贡献更小。
4. **可计算性**: 在最小实验中应存在可行近似，如 diagonal / block / fullized 版本。

### 候选定义

#### 候选 A: PI 加权 Fisher

`PIM_F = E_x [ π(x) · F_x ]`

其中 `F_x` 是样本级 Fisher 矩阵。

含义：PI 决定样本是否值得纳入结构估计，Fisher 决定结构方向。

#### 候选 B: PI 加权二阶矩代理

`PIM_v = E_x [ π(x) · diag(v_x) ]`

其中 `v_x` 是 Adam / ARS2-Neo 风格的梯度平方 EMA 代理。

含义：这是最便宜、最工程化的近似形式，适合最小实验。

#### 候选 C: PI 加权蒸馏偏差矩阵

`PIM_Δ = E_x [ π(x) · g_Δ(x) g_Δ(x)^T ]`

其中 `g_Δ(x) = ∇θ D(q_teacher(x), pθ(x))`。

含义：直接用 teacher-student 分布偏差作为结构张量来源，更贴近持续学习中的“保旧”目标。

#### 候选 D: 输出重要性-代价组合矩阵

令：

- `Λ_x` 为输出空间重要性矩阵
- `S_x = J_x^T Λ_x J_x` 为输出重要性拉回参数空间后的结构矩阵
- `C_x` 为参数空间认知代价矩阵

则 PIM 的两个更强候选形式为：

`PIM_1(x) = C_x^(1/2) S_x C_x^(1/2)`

`PIM_2(x) = S_x ⊙ C_x`

其中：

- `PIM_1` 强调几何一致性，适合理论分析
- `PIM_2` 强调工程可实现性，适合最小实验

这一候选把“输出偏离结构”与“认知代价结构”明确拆开，避免把二者过早混成单一梯度量。

### 输出重要性矩阵 `Λ_x` 的近似方案

对于候选 D 中的 `Λ_x`，直接计算输出对数似然的 Hessian (`H_z`) 成本过高。在最小实验中，我们需要寻找 `Λ_x` 的代理：

**方案 1：样本级梯度外积 (Rank-1 近似)**

`Λ_x ≈ r_x r_x^T`，其中 `r_x = ∂ℓ(x)/∂z` 是 loss 对 logits 的梯度。

这代表了当前样本*最急需修正的单一输出方向*。优点是计算极简，缺点是可能丢失其他非主导方向的重要性。

**方案 2：基于 Teacher 预测概率的对角矩阵**

`Λ_x ≈ diag(q_teacher(x))`

这假设高概率输出类别对微小扰动更敏感。这是许多知识蒸馏中隐含的结构假设。

**方案 3：基于 JVP 的隐式操作**

不显式实例化 `Λ_x` 或 `S_x`，而是通过雅可比向量乘积 (JVP) 隐式计算 PIM 作用于向量 `v` 的结果：

`S_x v = J_x^T (Λ_x (J_x v))`

这允许我们使用更复杂的 `Λ_x` 而不必承受巨大的内存代价。

### Invariants

1. 若 `π(x) = 1`，PIM 退化为原始结构矩阵的普通期望。
2. 若 `π(x) = 0`，该样本对结构保持无贡献。
3. 若采用各向同性高斯假设，则 `PIM ∝ I`，结构信息退化为标量尺度。
4. 若 PIM 无法比 scalar PI 提供额外方向信息，则其理论增益不成立。

### Proof Sketch

标量 PI 只能控制 `q_mix` 的整体强度，不能表达方向性，因此无法区分“哪些方向必须保持、哪些方向允许重写”。只要结构保持问题不是各向同性的，标量 PI 就必然信息不足。矩阵级对象之所以必要，是因为知识保持本质上依赖于方向敏感性，而方向敏感性只能由二阶对象表达。

## 最小实验方案

### 目标

不直接做完整 sequential training，而先验证一个更小的问题：

**在固定 teacher / student / 数据批次下，PIM 是否比 scalar PI 更能区分“结构敏感方向”与“非敏感方向”。**

### 建议实验形态

1. 使用当前已有的小 Transformer 与模运算 toy task。
2. 固定一个 teacher 和一个 student 初态。
3. 在一批样本上同时计算：
   - scalar PI
   - diagonal 二阶矩代理
   - 候选 D 中基于梯度外积近似的 PIM 对角线: `diag(S_x) ⊙ diag(v_x)`
4. 比较这些对象对参数更新方向的排序差异。

### 最小观测指标

- 各参数块的敏感度排序是否稳定。
- scalar PI 与 PIM 是否给出不同的方向偏好。
- 使用 PIM 加权后，更新是否更偏向少数高结构敏感方向。

## 决策

### 建议

首个 PIM 原型优先实现 **候选 D 的按元素重加权对角版本**，即：

`PIM_diag = E_x [ diag(S_x) ⊙ diag(v_x) ]`

其中 `diag(S_x)` 采用 rank-1 梯度外积近似拉回的对角线元素。

理由：

1. 这是真正将“输出偏差拉回”与“认知代价”结合的最简形式。
2. 标量 PI 乘二阶矩 (候选 B) 并没有真正引入输出空间的定向结构信息。
3. 通过对比 `PIM_diag` 和单纯的 `diag(v_x)` (原二阶矩)，能最清晰地展示拉回结构 `S_x` 的增益。

### 理由

- 标量 PI 解决的是“资格”问题，不是“结构”问题。
- Fisher/二阶矩解决的是“方向”问题。
- PIM 的核心思想正是把二者乘起来，而不是互相替代。
