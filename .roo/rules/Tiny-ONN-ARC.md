# 技术日志：Tiny-ONN-ARC 思想演化史 (v0.11.0)

本文档旨在以最高信息密度记录 `Tiny-ONN-ARC` 项目在攻克 ARC 任务过程中的核心思想迭代、关键技术范式及其演化路径。

## 1. 核心概念词汇表

按时间顺序排列，定义了项目演化中的关键范式与技术。

- **Teacher Forcing (v0.5)**
    标准的自回归训练范式。通过向模型提供完整的正确序列 `[Input, Output]`，并要求其仅预测 `Output` 的下一个 token。此方法因“暴露偏差”而失败：模型学会了依赖正确上下文的局部模式匹配（例如“前一个像素是蓝色，下一个也是”），而非真正的全局推理。在自回归生成时，微小的错误会迅速累积，导致“模式崩溃”。

- **EAVI (v0.6)**
    `Excursion-Alignment Variational Inference`（执行对齐变分推断）。为解决 Teacher Forcing 的短视问题而提出的对齐范式。其核心是强迫模型在训练中直面**独立生成**完整序列（认知远足）所带来的后果，然后将这个可能错误的完整生成与**全局真值**进行比较（对齐）。`main_loss` 因此成为一个全局的、无法作弊的信号，在理论上等同于最小化系统的变分自由能 (VFE)。灵感来自DPO。

- **OFL (v0.7)**
    `Object Finder Layer`（对象发现层）。为解决 EAVI 训练崩溃而设计。我们诊断出崩溃源于“孔径问题”：像素级路由缺乏宏观上下文。OFL 是一个置于模型前端的、非因果的全局 `DynSMHALayer`，其唯一任务是计算像素间的亲和度，将像素动态分组为“对象”，并将对象的全局上下文（原型）广播回每个像素。这实现了“感知”与“推理”的解耦，为模型提供了强大的结构归纳偏置。

- **Federal EAVI (v0.8)**
    联邦 EAVI，最终的优化范式。我们发现，标准的批次聚合更新 `gating_loss(grad(mean(L)))` 从根本上破坏了 EAVI 的元学习目标，因为它让门控去学习一个平均化的“幽灵任务”的路由。联邦 EAVI 通过**逐序列**立即执行 `backward()` 和 `optimizer.step()`，确保门控从清晰、独立的真实任务梯度 `mean(gating_loss(grad(L)))` 中学习，保留了元学习循环的即时性与信息保真度。

- **MoE Mismatch (v0.8.x)**
    MoE 架构的根本性错配。即使拥有了理论正确的 EAVI 优化，模型性能依然不佳。我们最终意识到，MoE 的归纳偏置是“从一个固定的专家库中**选择**一个变换来应用”，而 ARC 的本质是“从样本中归纳出原子规则并**组合**成一个新程序来执行”。MoE 试图用一个固定的专家库去拟合一个需要动态构建计算图的任务，这是失败的根源。

- **SparseBNN (v0.9)**
    `Sparse Bayesian Neural Network`（稀疏贝叶斯网络），“可微程序搜索”范式的具体实现。我们彻底放弃 MoE，将整个网络视为一个巨大的、潜在的程序空间。神经元被视为原子指令，而贝叶斯权重 `(μ, σ)` 则代表了指令间的连接通路及其确定性。训练过程通过从权重分布中随机采样来**探索**不同的计算图（程序实例），并通过 VFE 损失更新所有权重分布，从而实现梯度下降式的**程序搜索**。

- **Sequence-to-Image Fallacy (v0.10)**
    序列到图像的谬误，我们所有失败的“原罪”。我们最终认识到，将 ARC 这个本质上的二维空间推理问题，强行塞进一维自回归生成的“窄门”是根本性错误。这必然导致不可微的 `argmax` 操作，并人为地制造了训练（Teacher Forcing）与推理（自回归）之间的鸿沟。所有复杂的对齐范式，都只是在这个错误假设上试图弥合本不该存在的裂缝。

## 2. 思想演化路径

1. **v0.5 -> v0.6**: 意识到 `Teacher Forcing` 的局部信号无法驱动全局推理，转向 `EAVI` 范式，强制模型进行完整的自回归生成与全局对齐。
2. **v0.6 -> v0.7**: `EAVI` 训练崩溃，诊断为“孔径问题”，引入 `OFL` 架构，解耦“结构感知”与“规则应用”。
3. **v0.7 -> v0.8**: 训练仍不稳定，发现批次聚合更新破坏了 `EAVI` 的元学习，进化为 `Federal EAVI`，采用逐序列即时更新。
4. **v0.8 -> v0.9**: 即使优化正确，`MoE` 的“选择专家”归纳偏置与 ARC 的“程序合成”本质存在根本性错配，转向 `BSNN` 范式，进行“可微程序搜索”。
5. **v0.9 -> v1.0 (当前)**: 意识到所有基于自回归的尝试，都是在用一维方案解决二维问题。这是“序列到图像的谬误”，是所有问题的根源。

## 3. 当前范式：回归 Hyper-SMoE Vision Transformer

基于最终的领悟，我们决定彻底放弃自回归生成，回归到一个更符合问题本质的架构：**基于 Encoder-Decoder 的 Hyper-SMoE Vision Transformer (ViT)**。

这次“回归”是螺旋式的上升，而非简单的倒退。它建立在之前所有失败的洞察之上：

1. **并行化与二维感受野**: ViT 架构将图像作为并行的 patch/token 序列处理，拥有天然的二维感受野，完全规避了自回归的计算开销和理论谬误。
2. **保留核心创新**: 我们保留了 `DynSMHA` 和 `DynMoE` 的核心思想，让每个 Transformer Block 都成为一个完整的 `DynONN Block`，从而在 ViT 框架内实现动态的、token 级的计算路由。
3. **务实的工程决策**: 我们暂时放弃了理论优雅但计算昂贵且不稳定的 `SurpriseMinLoss` (SML)，转而采用原版 `DynMoE` 中经过验证的、更稳定的手工辅助损失函数（负载均衡、多样性等）。这使我们能更快地迭代和训练，首先确保核心架构的稳定与可行性。
