# PLSD: Per-Layer Speculative Decode

## 1. 架构定义与核心使命

`PLSD` 是 `Recursive DynSIHA` 的动态深度控制机制，旨在替代基于强化学习的 `ACT` 范式。它将计算资源分配问题从「动作探索」重构为「认知状态对齐」，通过自监督方式使模型感知自身的认知饱和点。

## 2. 形式化描述

在训练阶段，模型展开至最大递归深度 `T_max`，通过全量观测定义 `Oracle` 步长：

1. **损失序列生成**：
   对于输入 `x` 与标签 `y`，计算每一递归步的解码损失：
   `ℓ_t = CrossEntropy(LMHead(h_t), y)`, `t ∈ [1, T_max]`

2. **Oracle 步长 (认知饱和点)**：
   定义该样本在当前参数下的最优认知深度 `t*`：
   `t* = argmin_t ℓ_t`

3. **自监督对齐损失 (SS-SFT)**：
   训练停止头 `H(h_t)` 拟合以 `t*` 为阈值的阶跃函数：
   `Target_t = 0 if t < t* else 1`
   `ℒ_ACT = Σ_t BCE(H(h_t), Target_t)`

## 3. 泛化性能假设：RL vs. SS-SFT (PLSD)

`PLSD` 与传统 `Q-Learning ACT` 在泛化逻辑上存在本质区别：

- **RL 假设 (Q-Learning)**：泛化源于对状态-动作空间的广泛探索与贝尔曼自举。其假设价值函数在未知领域具有平滑性。但在 `ARC` 等离散逻辑任务中，稀疏奖励导致探索效率极低，易陷入局部最优或始终跑满深度。
- **SS-SFT 假设 (PLSD)**：泛化源于对「最优认知轨迹」的确定性拟合。它假设模型内部已存在解决任务的潜力，`PLSD` 的职责是提取该潜力的「最短路径特征」。它通过 `Teacher Forcing` 的时间扩展，将动态退出转化为对 `Loss` 地形饱和点的模式识别。

## 4. 优化器耦合与风险控制

由于 `PLSD` 本质上是强监督拟合 `Oracle` 目标，其泛化能力高度依赖于参数空间的拓扑质量：

1. **风险**：单纯的 `SS-SFT` 极易导致停止头 `H(h_t)` 对特定训练样本的计算路径过拟合，失去对异构复杂度的外推能力。
2. **ARS2-Neo 协同**：`PLSD` 并不自带泛化保证，它必须依赖 **ARS2-Neo 优化器** 提供的 **SAM (Sharpness-Aware Minimization)** 风格的平坦度约束。
3. **机制**：`ARS2` 通过 `Fisher` 信息近似与能量-几何解耦，强制 `H(h_t)` 的决策边界位于损失平面的平坦区域。这种「平坦决策」确保了模型对「何时停止」判断不依赖于微小的表征扰动，从而实现稳健的任务外推。
